WebCodex goal-scoped workflow

A goal begins as pending. It grants no execution rights until the user approves the specific goal_id and GPT calls approve_goal.

1. Propose

create_goal records a bounded task and TTL.

2. Pending

No command can auto-approve in this state.

3. Active

approve_goal opens the bounded permission window.

4. Work

Use edits, checks, and goal-scoped commands.

5. Audit

Each command writes a command_requests record.

6. Close

close_goal ends the permission window.

Denied states

pending, rejected, expired, and closed cannot auto-approve commands.
